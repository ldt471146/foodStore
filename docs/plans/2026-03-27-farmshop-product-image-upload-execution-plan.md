# 农场商品图片上传执行计划

## 执行级别

M 级，单链路改造。

## 实施范围

1. 后端商品管理接口改为接收 `multipart/form-data`。
2. 后端新增商品图片保存与读取能力。
3. 前端管理后台删除 URL 输入，改为文件上传和预览。
4. 保持前台消费侧继续读取 `imageUrl`。

## 实施步骤

### 1. 后端接口改造

- 调整 `AdminController` 商品新增/编辑接口签名。
- 将商品表单字段改为 `@RequestParam` 接收。
- 增加商品图片读取接口。

### 2. 后端服务改造

- 在 `AdminService` 中增加商品图片保存逻辑。
- 编辑商品时未上传图片则保留旧图。
- 删除商品时不强制清理历史图片文件，避免误删共享资源。

### 3. 前端管理后台改造

- 删除 `imageUrl` 文本输入。
- 新增本地图片文件选择、预览、编辑态保留旧图。
- 通过 `FormData` 提交新增/编辑商品。

### 4. 验证

- 后端执行 `run-mvn-jdk17.cmd -q -DskipTests compile`
- 前端执行 `npm run build`

## 回滚规则

- 若 `multipart/form-data` 影响商品保存，优先回滚到原 `imageUrl` JSON 提交链路。
- 若前端上传逻辑异常，保留后端读取接口代码不动，单独回滚表单提交实现。

## 清理要求

- 不生成临时测试资源文件。
- 不启动服务进程。
