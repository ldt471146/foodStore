# 农场品在线商城设计说明

## 方案比较

### 方案 A：严格复刻开题报告的 SSM + Vue + MySQL + Python 算法
- 优点：与报告文字最一致
- 缺点：搭建和联调成本高，IDEA 本地运行链更长，Python 算法服务会增加演示脆弱性

### 方案 B：Spring Boot 单体后端 + Vue 3 前端 + 双数据库配置
- 优点：本地运行简单，适合 IDEA；保留清晰分层；后续扩展到 MySQL 很直接
- 缺点：与报告里的 `SSM` 用词不完全一致，需要在答辩时解释为工程升级

### 方案 C：前后端全前端模拟 + 本地假数据
- 优点：开发速度最快
- 缺点：无法满足后端业务管理和答辩可信度要求

## 选型结论
采用方案 B。它能最大化兼顾“可交付代码量”“IDEA 可运行性”“全栈演示完整度”三个目标。

## 系统结构
- `backend/`：Spring Boot REST API，负责鉴权、业务规则、数据持久化、种子数据
- `frontend/`：Vue 3 SPA，负责商城前台与管理后台页面
- 数据层采用关系模型，核心实体包括用户、商品、分类、溯源、购物车、订单、订单项、库存流水、评价、浏览记录、物流记录

## 页面结构
- 商城端：首页、商品列表、商品详情、购物车、我的订单、推荐与评价
- 管理端：控制台、商品管理、库存管理、订单处理、客户分析、销售预测

## 智能模块设计
- 推荐：以浏览记录和购买记录计算偏好标签，生成基于分类与热度加权的推荐列表
- 预测：使用历史订单销量按月聚合后做趋势平滑预测，输出 ECharts 所需序列
- 分析：对订单和库存数据做聚合，生成排行、预警与摘要指标

## 风格落地
- 统一使用自然大地色、圆润胶囊输入框、柔和卡片、低饱和绿色/棕色点缀
- 使用 Tailwind 自定义 token，把风格包颜色与圆角写入主题变量
- 管理后台避免传统冰冷表格风，保留自然有机风的卡片和图表布局

## 运行方式
- 后端在 IDEA 中作为 Maven Spring Boot 项目启动
- 前端在 IDEA Terminal 中执行 `npm install` 与 `npm run dev`
- 默认后端端口 `8080`，前端端口 `5173`
