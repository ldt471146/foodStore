# 农场品在线商城入口与权限重构执行计划

## Internal Grade
`L`

## Wave Structure

### Wave 1：入口重构
- 把根路由改为登录入口
- 把业务内容路由统一改为登录后访问
- 增加登录后按角色跳转逻辑

### Wave 2：权限表达
- 收紧前端守卫
- 收紧后台页签显示
- 保持后端 `/api/admin/**` 角色限制

### Wave 3：视觉重构
- 重做登录页
- 引入本地中国果园插画背景
- 把登录页从公共布局中拆出，避免未登录时泄露业务导航

### Wave 4：覆盖清单与收尾
- 对照 [功能实现.md](/D:/aigent/zhouri/功能实现.md) 输出覆盖清单
- 补齐库存报表可视化页签
- 记录本轮 vibe phase artifact
- 不做运行结论，只交付代码与待用户本地验证清单

## Verification Commands
- 本轮按用户要求不主动运行
- 由用户在 IDEA 中自行验证启动与页面跳转

## Rollback Rules
- 若登录入口导致前台内容不可达，优先检查路由守卫和默认跳转
- 若管理员功能不可见，优先检查角色判断和后台页签过滤
- 若登录页背景失效，优先检查本地静态资源导入路径

## Cleanup Expectations
- 保留本轮需求冻结文档、执行计划、功能覆盖清单
- 记录 phase receipt 与 cleanup receipt
- 不删除现有业务页面
