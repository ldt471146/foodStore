## 项目目标
构建一套可在 IDEA 中直接运行的“农场品在线商城”全栈演示系统，覆盖消费者端与管理端核心业务链路，并保留毕业设计答辩所需的智能化模块展示能力。

## 技术决策
- 后端：Spring Boot 3、Spring Web、Spring Security、Spring Data JPA、Validation、JWT
- 数据库：默认 `H2` 开箱即跑；保留 `MySQL` 配置入口，便于答辩或后续切库
- 前端：Vue 3、Vite、TypeScript、Pinia、Vue Router、Axios、Tailwind CSS、ECharts
- 算法实现：先以 Java 服务内置规则与时间序列近似预测完成可演示功能，不引入额外 Python 运行链，保证 IDEA 一键启动稳定性
- 运行组织：前后端分仓目录但同一项目根目录，便于在 IDEA 中分别启动

## 角色
- 消费者：注册、登录、浏览商品、加入购物车、下单、支付、评价、查看物流、找回密码
- 农场管理员：商品管理、库存维护、订单处理、物流录入、客户查看、报表查看
- 平台管理员：全局用户查看、商品与订单总览、销售分析、预警总览

## 必做功能
- 用户管理：注册、登录、退出、角色区分、找回密码
- 商品管理：列表、分类筛选、详情、溯源信息、后台 CRUD
- 订单管理：购物车、提交订单、模拟支付、订单状态跟踪、后台处理
- 库存管理：库存数量、入库出库记录、低库存预警、库存报表
- 客户管理：用户基本信息、历史购买记录、客户画像摘要
- 推荐功能：基于浏览与购买记录生成个性化推荐
- 销售预测：基于历史销量生成未来趋势图
- 数据分析：销量统计、热销排行、库存预警摘要
- 评价与物流：商品评分评价、物流信息录入与查询

## 非目标
- 不接入真实支付网关
- 不接入短信、人脸识别或真实物流平台
- 不做分布式部署、消息队列或对象存储
- 不追求完整生产级风控，仅保证演示级安全与数据隔离

## 设计约束
- 页面风格必须遵循 `natural-organic` 色板与自然有机视觉语言
- 前端组件优先自定义，避免 UI 库默认风格破坏视觉一致性
- 所有核心模块都要可通过界面直接演示，不接受只存在数据库或只在控制台输出
- 默认启动方式必须适配 IDEA：后端直接运行 Spring Boot 主类，前端运行 `npm run dev`

## 验收标准
- IDEA 中启动后端成功，接口可返回种子数据
- 前端页面可正常访问消费者首页、商品详情、购物车、订单页、管理后台
- 注册登录后可完成浏览商品、加入购物车、提交订单、模拟支付、查看物流、提交评价
- 管理后台可完成商品 CRUD、库存调整、订单状态推进、物流单号录入
- 分析页能展示销量排行、库存预警、销售趋势图、推荐列表
