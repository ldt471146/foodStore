# 评论权限与商品发布人需求

## 目标

完善评论系统与商品发布信息展示，使评论行为与真实购买记录绑定，并让商品发布人能够查看该商品收到的评论。

## 范围

- 只有购买并完成支付的用户才能评论商品。
- 未购买商品的用户在前端看到明确提示，后端仍然强制拦截。
- 我的订单中每个商品都提供单独评论入口。
- 商品列表、商品详情等位置展示商品发布人。
- 商品发布人可以在管理后台看到用户对自己商品的评论。

## 约束

- 保持现有 Spring Boot + MyBatis 架构。
- 保持当前“一人一商品一条评论”规则，不扩展为多条评论。
- 不启动项目，只做代码修改与编译构建验证。

## 验收标准

- 未购买用户在商品详情页无法提交评论，并看到“购买后才能评论”提示。
- 已购买且已支付用户可在订单页对订单中的商品评论。
- 已评论商品在订单页显示已评价，不重复出现可提交入口。
- 商品详情页能看到发布人信息。
- 农场管理员在管理后台只能看到自己发布的商品，并能看到这些商品的用户评论。
