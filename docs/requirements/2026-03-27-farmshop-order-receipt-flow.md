# 订单送达与确认收货需求

## 目标

重构订单流转，区分“农场管理员标记已送达”和“消费者确认收货”两个动作，避免“已送达”与“已签收”混在一起。

## 订单状态

- `处理中`：消费者支付完成，等待发货。
- `已发货`：农场管理员录入物流信息后进入。
- `已送达`：农场管理员确认物流送达后进入。
- `已收货`：消费者在订单中心点击“确认收货”后进入。

## 验收

- 管理端保存物流后，订单进入 `已发货`。
- 管理端可以将已发货订单标记为 `已送达`。
- 消费者只能对 `已送达` 的订单点击“确认收货”。
- 消费者确认收货后，订单状态变为 `已收货`。
- 前后端状态文案全部为中文，且页面能看到各阶段时间与提示。
