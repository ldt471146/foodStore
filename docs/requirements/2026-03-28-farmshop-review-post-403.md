# 农产品商城评论提交 403 与订单级评论资格需求冻结

## 来源

- 用户提供的压缩排查结论：`POST /api/catalog/products/1/reviews` 在同一 token 下独占返回 `403`
- 关联冻结需求：`docs/requirements/2026-03-27-farmshop-multi-review.md`

## 目标

- 定位并修复评论提交接口 `POST /api/catalog/products/{productId}/reviews` 的独占 `403`
- 将评论资格从“历史上买过且收货过该商品即可”收口到“针对当前已确认收货的订单项才能评论”
- 保持“同一商品允许多次追加评论”的既有目标

## 约束

- 不启动用户本地前后端服务
- 优先通过后端测试和静态代码分析锁定根因
- 修改需覆盖前后端接口契约，避免仅修前端提示

## 验收标准

- 后端测试能覆盖“同 token 下 browse 可用、reviews 可用”的接口行为
- 评论提交不再依赖“历史任意收货订单”判定
- 订单页只对已确认收货的当前订单项开放评论入口
- 用户自行重启后，可直接复测评论提交链路
