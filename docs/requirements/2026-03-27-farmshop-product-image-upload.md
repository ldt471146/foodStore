# 农场商品图片上传需求

## 目标

将农场管理员商品管理中的商品图片维护方式，从手工填写 `imageUrl` 文本链接，调整为本地图片文件上传。

## 当前问题

- 管理后台商品表单仍要求输入图片链接，操作门槛高。
- 农场管理员无法直接从本机选择图片上传。
- 用户已经明确要求删除商品 URL 录入方式。

## 目标交付

- 管理后台新增商品时可直接上传商品图片。
- 管理后台编辑商品时可更换商品图片，也可不上传并保留原图。
- 后端将商品图片保存到本地目录，并通过接口回显图片。
- 前台首页、列表页、详情页继续通过 `imageUrl` 字段显示图片，不额外改动展示模型。

## 约束

- 不启动项目，只改代码并做编译/构建验证。
- 保持 Spring Boot + MyBatis 现有架构，不引入 JPA 仓储。
- 不新增外部对象存储服务。
- 不改变数据库 `products.image_url` 字段语义，仍用于保存可访问图片地址。

## 验收标准

- 农场管理员商品表单中不再出现“图片链接”输入框。
- 商品表单提供文件上传入口与图片预览。
- 后端新增/编辑商品接口支持 `multipart/form-data`。
- 成功保存商品后，商品列表和前台页面可继续显示上传后的图片。
- 后端编译通过，前端构建通过。
